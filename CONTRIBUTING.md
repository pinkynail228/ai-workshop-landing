# Правило Работы

Коротко: `1 промт = 1 коммит = 1 push в main`.

## Как работаем

1. Внес изменение по одному промту.
2. Проверил локально (`python3 -m http.server 8123`).
3. `git add -A`
4. `git commit -m "коротко что изменил"`
5. `git push origin main`

## Что это дает

1. Каждое изменение зафиксировано отдельно.
2. Легко откатиться к любой точке.
3. После `push` в `main` Vercel автоматически обновляет `https://ailending.vercel.app`.

## Важно помнить

1. Локальный `commit` сам по себе не деплоит, деплой начинается после `push`.
2. Не коммитим `.vercel/` и `.DS_Store`.
3. Если надо откатить одно изменение: `git revert <commit_sha>` и `git push origin main`.
